## Codebase Patterns
- Keep Watchtower's Ralph control files (`AGENTS.md`, `CLAUDE.md`, `CODEX.md`, `progress.txt`, `prompt.md`, `prd.json`) together under `watchtower/`.
- Watchtower development happens directly on `main`; do not create separate Ralph branches for stories.
- Commit messages should read like normal developer commits and stay all lowercase.
- Treat `Shadowbroker/` as a separate local reference codebase and ignore it from the Watchtower repo root.
- Use npm workspaces from the repo root to manage `frontend/` and `backend/`; root `build`, `lint`, and `test` scripts are the standard validation entrypoints.
- Keep frontend-to-backend development traffic behind the Vite proxy on `/api`, with baseline shell health exposed from backend `/api/status` and `/health`.
- The current frontend map shell defines its layer UI and placeholder overlay points side by side in `frontend/src/App.tsx`; keep `layerDefinitions` and `layerMarkers` aligned until live data moves into a shared store.
